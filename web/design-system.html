مدل دادهٔ فایربیس

ساختار پیشنهادی برای Cloud Firestore. داشبورد و صورت‌حساب فقط‌خواندنی‌اند؛ تنها فرم «افزودن عضو» در مجموعهٔ members می‌نویسد. مبالغ به‌صورت عدد صحیح (تومان) ذخیره می‌شوند و قالب‌بندی فارسی هنگام نمایش اعمال می‌گردد.

members/{memberId}
  firstName: string          // required
  lastName:  string          // required
  family:    string
  dob:       timestamp | null
  phones:    string[]        // 0..n  — repeatable
  accounts:  string[]        // 0..n  — repeatable
  referredBy: memberId | null // معرف — who introduced them
  status:    "active" | "inactive"
  loanReceived: boolean      // received a loan THIS round
  loanPos:   number          // position in the loan order
  behind:    boolean
  missed:    number
  createdAt: serverTimestamp

  shares/{shareId}           // sub-collection — one doc per share line
    label:     string        // "سهم الف" …
    since:     timestamp
    balance:   number        // AUTHORITATIVE stored value (Rials) — never
                             // recomputed from the fee. Incremented when a
                             // payment is recorded; survives fee changes.
    funding:   boolean       // true while still funding up to par
    // funded      = balance >= fund/config.parValue   → loan-eligible
    // fundedPct   = balance / parValue                → drives the progress bar

  loan/{loanId}              // 0..1 active loan (only on a FUNDED share)
    principal:        number
    term:             number // months
    installmentsPaid: number
    monthly:          number
    outstanding:      number

  payments/{paymentId}       // ledger — every membership fee / installment
    type:    "membership" | "installment"
    shareId: string | null   // installment target
    amount:  number
    date:    timestamp
    // recording a payment does: shares/{id}.balance += amount

fund/config                  // single settings doc — INFORMATIONAL only
  membershipFee:       60    // monthly fee (drives no balance math)
  defaultInstallments: 20    // default installments for a new share
  parValue:            5780   // current full-funded value of a share
  // parValue rises by membershipFee each cycle (next month → 5840)
  asOf:                timestamp

fund/loanRotation            // round-based loan order (admin-managed)
  round:  number             // current round number
  order:  memberId[]         // the loan queue — drag to reorder
  // members keep their place; loanReceived flags them. Next-up = first
  // member in order with loanReceived == false. When all have received,
  // round++ and every loanReceived resets to false.

موجودی هر سهم یک عدد ذخیره‌شدهٔ معتبر است که فقط هنگام ثبت پرداخت بالا می‌رود — هرگز از روی حق عضویت محاسبه نمی‌شود، پس با تغییر حق عضویت دستکاری نمی‌شود. یک سهم تا وقتی به ارزش کامل (parValue) تأمین نشود، واجد شرایط وام نیست. دسترسی اعضا به صورت‌حساب خودشان با Security Rules به request.auth.uid == memberId محدود می‌شود.
